🎯 **What:** The pure React functional component `FloatingNav` within `src/renderer/components/chrome.jsx` was untested, leaving a critical portion of the primary application navigation unverified.

📊 **Coverage:** Added a comprehensive suite of jsdom-based unit tests for `FloatingNav` in `tests/chrome.test.jsx`. The tests now verify:
- Accurate rendering of the correct navigation items ("Servers", "Console", "History", "Settings").
- The proper assignment of the `.active` class based on the currently selected `view` prop.
- The `setView` handler triggers correctly when navigating.
- The `selectedCount` badge appears properly on the "Console" item when servers are active, and is absent when the count is zero or null.
- Correct execution of the `onAddServer` and `onToggleTheme` prop functions.

✨ **Result:** Test coverage significantly increased for the core UI shell. Future refactoring of the FloatingNav or additions of new navigation items can now be verified deterministically against regressions.
